Instalação do ambiente
Versão testada Python 3.11.14

1º Instalar ambiente virtual
    sudo add-apt-repository ppa:deadsnakes/ppa
    sudo apt update
    sudo apt install -y python3.11 python3.11-venv

2º Criar ambiente
    /usr/bin/python3.11 -m venv /home/rz/venv3_11_14

3º Ativar ambiente
    source /home/rz/venv3_11_14/bin/activate

4º Instalar Driver NVIdia 
    sudo ubuntu-drivers autoinstall    
    sudo reboot

5º Adicionar cuda 12.8
    wget https://developer.download.nvidia.com/compute/cuda/repos/ubuntu2404/x86_64/cuda-ubuntu2404.pin
    sudo mv cuda-ubuntu2404.pin /etc/apt/preferences.d/cuda-repository-pin-600

    wget https://developer.download.nvidia.com/compute/cuda/12.8.0/local_installers/cuda-repo-ubuntu2404-12-8-local_12.8.0-570.86.10-1_amd64.deb

    sudo cp /var/cuda-repo-ubuntu2404-12-8-local/cuda-*-keyring.gpg /usr/share/keyrings/

    Instalar o REPO
        sudo dpkg -i cuda-repo-ubuntu2404-12-8-local_12.8.0-570.86.10-1_amd64.deb

    Instalar variaveis ambiente
        export PATH=/usr/local/cuda-12.8/bin${PATH:+:${PATH}}
        export LD_LIBRARY_PATH=/usr/local/cuda-12.8/lib64${LD_LIBRARY_PATH:+:${LD_LIBRARY_PATH}}


    Verificar Instalação
        nvcc --version


Resumo do Serviço
Objetivo deste microserviço é gerar um banco com embbedings para comparar textos através de similaridade
Para se gerar o dataset necessário é necessario seguir os passos a seguir

Objetivo macro do Algoritimo

1º primeiro popular um banco de dados vetorial com embbedings baseado na tabela textos_treinamento

2º Uma vez populada a tabela o sistema procura Ids altamente semelhantes para não criar duplicadas na collection do banco vetorial, estes ids tem a mesma classe porem
    o texto é altamente semelhante

3º O sistema procura ids que estão classificados errados , o conceito é procurar textos muito parecidos com classes diferentes dando a opção pro usuario
    reclassificar estes ids

4º Após isso o usuario deve corrigir os textos em ids colidentes

5º deve se popular o banco de dados com textos_classificar sem que estes dados sejam previamente treinados em textos_treinamento aplicar os algoritimos para classificar
    corrigir os resultados


Passos para execução do sistema
1º Gerar embbedings através do endereço http://10.1.1.95:5020/generateembeddings
    Como dica de otimização a primeira que é gerado se o banco tiver muitos idsiguais pode se chamar api para ela e começar denovo pois o sistema só ira processar textos unicos

2º Encontrar ids altamente semelhantes através do endereço  http://10.1.1.95:5020/generate_ids_iguais

3º Encontrar ids colidentes ou seja altamente conflitantes que tenham algo grau de similaridade chamando a api  http://10.1.1.95:18020/generate_ids_colidentes
